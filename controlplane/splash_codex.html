The security trifecta, plus policy gates

1. Hard sandbox boundaries

Each agent runs in an isolated VM with only the files and integrations you explicitly attach.

2. Encrypted secrets broker

API keys stay out of the agent context. Credentials are injected at call time so models never see raw secrets.

3. Outbound allowlist + exfil monitoring

Network egress is monitored and constrained by policy, reducing accidental leakage and hostile prompt chains.

4. Integration policy gates

Gmail/WhatsApp/other tools can be restricted to allowed actions, senders, subjects, and response formats.

5. Best-practice platform controls

OAuth, encryption, and minimal personal data retention (names + email only) with auditable control-plane enforcement.
